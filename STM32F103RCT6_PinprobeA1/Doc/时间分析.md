# 状态机时间分析

## 计时基准

- **TIM1**：1ms 周期定时器（72MHz / 72 / 1000 = 1kHz）
- **状态机调用周期**：~50ms（ModBusTask `osDelay(50)`）
- **计时方式**：基于 TIM1 毫秒时间戳，所有延时使用 `GetTim1Ms()` 绝对时间比较

## 延时参数

| 宏定义 | 值(ms) | 说明 |
|--------|--------|------|
| `LOCK_PRESS_MS` | 300 | 锁定按钮按下确认，需保持300ms才触发 |
| `LOCK_IDLE_MS` | 1000 | 锁定按钮冷却，上次操作1秒后才允许再次操作 |
| `DOOR_BUTTON_MS` | 300 | 门按钮基础防抖时间 |
| `DOOR_READY_MS` | 300 | Idle状态下门按钮按下确认时间 |
| `DOOR_CLOSE_CONFIRM_MS` | 1000 | Ready状态下需同时按双按钮1秒才关门 |
| `DOOR_OPEN_CONFIRM_MS` | 500 | Complete状态下按开门按钮500ms触发 |
| `RELEASE_DELAY_MS` | 200 | 门按钮释放后需持续松开200ms |
| `AIR_STABILIZE_MS` | 2000 | 关门后气压稳定等待2秒 |
| `AIR_CHECK_INTERVAL_MS` | 1500 | 气压检测间隔1.5秒 |

## 状态流转图

```
上电 ──→ SYS_INIT ──→ Lock ──→ Idle ──→ Ready ──→ Running ──→ Complete ──→ Idle
               ↑        ↑  ↓      ↑  ↓       ↑  ↓         ↑  ↓         ↓
               └────────┴──┘      └──┘       └────────────┘            ↑
               │                                                        │
               └── Emergency ←── 急停/激光/关门超时 ─────────────────────┘
```

## 各状态动作时序

### 1. Lock_Action（锁定状态）

```
                    ┌── LOCK_PRESS_MS(300ms) ──┐
                    │                          │
按下 power_button ──┼──────────────────────────┼── 触发 LOCK/UNLOCK
                    │                          │
        条件: lock_released == 1               │
              冷却时间已过 (LOCK_IDLE_MS 1s)    │
                                              │
触发后: lock_released = 0  ← 封锁后续触发      │
        lock_release_tick = now ← 启动冷却     │
                                              │
松开 power_button ──→ lock_released = 1  ← 解除封锁
```

**防重复机制**：
- `lock_released`：必须松开过才能再次触发（解决长按反复触发）
- `lock_release_tick`：两次操作间必须间隔 LOCK_IDLE_MS（防止快速点击）

### 2. Idle → Ready（空闲→准备）

```
按下 door_button1/2 ──→ door_ready_tick = now
                          │
                          ├── 持续按住 ──→ (now - door_ready_tick) >= DOOR_READY_MS(300ms)
                          │                 │
                          │                 ├── release_start_tick == 0 ✓
                          │                 │   黄灯亮 → system_status = Ready
                          │                 │
                          │                 └── release_start_tick != 0 ✗（释放未完成）
                          │
松开 ──→ door_ready_tick = 0（重置）
```

### 3. Ready → Running（关门开始）

```
同时按下 door_button1 AND door_button2
    │
    ├── door_close_confirm_tick = now
    │
    ├── 按住 >= DOOR_CLOSE_CONFIRM_MS(1000ms) → door_ready_status = 1
    │
    ├── release_start_tick == 0 ✓
    │
    └── 触发关门:
         Cylinder_Write(CLOSE)
         door_close_start_tick = now    ← 记录关门开始时刻
         door_close_timing = 1
         release_start_tick = now       ← 开始释放计时
         system_status = Running
```

### 4. Running → Complete（关门完成）

```
关门中 (out_01_08 & door_close)
    │
    ├── 限位传感器触发 (in_01_08 & door_sensor_down)
    │       │
    │       ├── door_close_default_ms = GetTim1Ms() - door_close_start_tick
    │       │                           ↑ 记录实际关门耗时
    │       ├── door_close_timing = 0
    │       ├── 绿灯亮
    │       └── system_status = Complete, door_status = Door_Closed
    │
    └── 未触发 → 继续 Running
```

### 5. Complete → Idle（开门完成）

```
开门中 (out_01_08 & door_open)
    │
    ├── 前限位触发 (in_01_08 & door_sensor_up)
    │       │
    │       ├── 关灯
    │       ├── door_open_start_tick 记录开门耗时 → LOG_ACTION_TIME("OPEN_DONE", ...)
    │       └── system_status = Idle, door_status = Door_Opened
    │
    └── 开门请求检测:
         按下 door_button → door_open_confirm_tick 开始
         按住 >= DOOR_OPEN_CONFIRM_MS(500ms) → door_complete_status = 1
         release_start_tick == 0 → Cylinder_Write(OPEN)
```

### 6. Release_detection（按钮释放检测）

```
两个 door_button 都松开:
    │
    ├── release_start_tick != 0 且持续松开 >= RELEASE_DELAY_MS(200ms)
    │       → release_start_tick = 0（释放完成，允许新操作）
    │
    └── 任一按钮仍按下:
            → release_start_tick = now（重置释放计时）
```

### 7. Emerge_Action（紧急检测）

#### 7.1 急停按钮

```
stop_button 按下 (!=0x08)
    → 立即 Emergency: 开门 + 解锁 + 红灯
```

#### 7.2 防夹手检测 (气压 + 激光, 关门中)

```
关门中 (door_close_timing=1, door_sensor_down=0)
    │
    ├── 前 2/3 时间: 不检测 (关门时间未学习也跳过)
    │
    └── 后 1/3 时间 (elapsed > door_close_default_ms * 2/3):
          任一触发:
           气压 (0x20) 或 激光2 (0x40) 或 激光3 (0x80) 或 激光4 (0x0100)
            → Emergency: 开门 + 解锁 + 红灯
            → door_close_start_tick = 0, door_close_timing = 0

  前置条件: door_close_time_learned = 1 (至少完成一次全开→关门学习)
```

#### 7.3 气压检测时序 (关门完成后)

```
关门完成 (door_sensor_down=1, door_close_done_tick != 0)
    │
    ├── elapsed < AIR_STABILIZE_MS(3000ms) → 气压稳定期，不检测
    │
    └── elapsed >= 3000ms:
         每隔 AIR_CHECK_INTERVAL_MS(1500ms) 检测一次
         气压不足 (0x20 != 1) → U1_Printf "Intake air pressure too low"
```

## 动作时间打印

定义 `ACTION_TIMING` 后，以下动作会通过串口输出耗时：

| 标签 | 时机 | 含义 |
|------|------|------|
| `[LOCK] xxx ms` | 锁定触发 | power按钮按下保持时长 |
| `[UNLOCK] xxx ms` | 解锁触发 | power按钮按下保持时长 |
| `[CLOSE_START] 0 ms` | 关门开始 | 标记（耗时为0） |
| `[CLOSE_DONE] xxx ms` | 关门完成 | 从开始到限位触发的实际关门耗时 |
| `[OPEN_START] 0 ms` | 开门开始 | 标记（耗时为0） |
| `[OPEN_DONE] xxx ms` | 开门完成 | 从开始到前限位触发的实际开门耗时 |
