概述：为什么用 AI + HTML 生成 PPT？
传统 PPT 痛点  |  AI 方案优势  |  适用场景
✗ 传统 PPT 痛点
  • 手动排版耗时，一套报告需数小时
  • 格式不统一，每次需重新调整样式
  • 数据变更后需逐页修改
  • 多人协作时版本混乱
  • 批量生成困难，无法自动化
✓ AI + HTML 方案
  • AI 从数据源自动填充内容，几分钟出报告
  • CSS 设计系统保证全报告风格统一
  • 数据源变更 → 重新生成即可，无手工调整
  • 单文件 HTML，版本可 Git 追溯
  • html2canvas 一键导出 PNG，批量自动化

核心理念

把 PPT 的「格式与样式」交给设计系统（template.html），把「内容生产」交给 AI，把「导出」交给脚本。人只负责审核与决策。

使用方式

AI 从 template.html 中复制对应页面的完整 HTML 代码块（从 <!-- Px ... --> 到 </div>），替换其中的占位文字为实际数据，即可生成一页完整 PPT。

如何编写 Prompt：给 AI 的指令模板
高质量 Prompt = 角色 + 模板引用 + 页面清单 + 数据源 + 规则约束
标准 Prompt 模板（5 要素）
要素说明示例
1. 角色设定明确 AI 的身份"你是一个PPT报告生成助手，使用GTS HTML设计系统..."
2. 模板引用指向 template.html"请先阅读 release/model/template.html，理解其CSS组件体系..."
3. 页面清单列出每页的类型和内容"P1 封面，标题《XXX》；P2 标准内容，表格数据来自 Excel..."
4. 数据源指定数据来源"从 BOM/data/xxx.xlsx 读取第1-10行，映射到表格..."
5. 规则约束限制格式和行为"每页不超过15行；P0用红色底色；子件缩进24px..."

关键硬规则
#规则说明错误示例 vs 正确示例
1CSS 必须完整复制新 HTML 的 <style> 必须包含 template.html 的全部 CSS。缺组件则显示异常。缺 .card → 卡片无样式 vs 完整复制
2页面固定尺寸.page 宽度 1120px、最小高度 630px。不可修改，否则打印裁切异常。width:auto → 错 vs width:1120px 正确
3内容区不溢出.main 中表格行数控制在 15 行以内。超过则分页或缩小字号。25行挤一页 → 溢出不美观
4页码必须连续每页 .page-num 必须是 "X / N" 格式，X 从 1 到 N 连续递增。2/3 → 3/5 → 不连续 vs 1/5 → 2/5 正确
5颜色语义一致红色=高风险/P0/外发 ；橙色=中风险/警告/P1 ；绿色=低风险/完成/通过 ；蓝色=信息/采购/待办.tag-r 标"已完成" → 语义错误 vs .tag-g 标"已完成"
6子件缩进一致子件行 td 第一个单元格用 padding-left:24px，名称列用 padding-left:8px。无缩进 → 层级不清
7字体不混用全部使用设计系统字体族：等线 → SimSun → Microsoft YaHei → sans-serif。不引入外部字体。font-family:"Arial" → 不一致
8图片路径正确logo 路径用相对路径 "../logo.png" 或 "../../release/logo.png"，检查文件层级。src="logo.png" → 路径错误

导出与发布
HTML → PNG（逐页）/ PDF（浏览器打印）
方式一：html2canvas 批量导出 PNG（推荐）
// 在 HTML 末尾 </body> 前加入以下脚本
<script src="https://cdn.jsdelivr.net/npm/html2canvas@1.4.1/dist/html2canvas.min.js"></script>
<script>
async function downloadAllPages() {
  const pages = document.querySelectorAll('.page');
  for (let i = 0; i < pages.length; i++) {
    const canvas = await html2canvas(pages[i], {
      scale: 2,              // 2x 高清输出（约 2240×1260）
      useCORS: true,         // 允许跨域图片
      backgroundColor: '#ffffff',
      logging: false
    });
    const link = document.createElement('a');
    link.download = `P${String(i+1).padStart(2,'0')}.png`;
    link.href = canvas.toDataURL('image/png');
    link.click();
    await new Promise(r => setTimeout(r, 300));  // 间隔 300ms 防浏览器阻塞
  }
}
// 在浏览器 Console 中调用：downloadAllPages()
</script>

// 也可以加一个按钮

最佳实践与注意事项
提升生成质量  |  减少返工
① 先少后多
  • 首次让 AI 生成 2-3 页验证风格
  • 确认样式无误后再扩展全量
  • 用「更新已有报告」指令增量修改
  • 避免一次性生成 20+ 页再排查
② 数据先行
  • Excel 数据整理好再喂给 AI
  • 表头清晰、无合并单元格
  • 避免 AI 猜测数据含义
  • 大数据集先做筛选和聚合
③ 模板即规范
  • template.html 是唯一样式来源
  • 不随意添加自定义 CSS
  • 新增样式需同步到 template
  • 颜色严格按语义使用
